# Plan 02-03: REST Endpoints for Data Delivery

## Objective
Create the REST controller layers inside the NestJS application to serve the homily dataset to the upcoming Next.js frontend frontend via HTTP.

## Tasks

<task>
<read_first>
- apps/api/src/app.module.ts
</read_first>
<action>
Generate a `HomiliesModule` comprising a Controller and a Service (`apps/api/src/homilies/homilies.module.ts`, `controller.ts`, `service.ts`) and import it into `app.module.ts`. Ensure CORS is enabled in `main.ts` to allow local interactions.
</action>
<acceptance_criteria>
- `apps/api/src/homilies/homilies.controller.ts` exists and has `@Controller('homilies')` decorator
- `apps/api/src/app.module.ts` contains `imports: [HomiliesModule]`
- `apps/api/src/main.ts` contains `.enableCors(`
</acceptance_criteria>
</task>

<task>
<read_first>
- apps/api/src/homilies/homilies.controller.ts
- apps/api/src/homilies/homilies.service.ts
</read_first>
<action>
Implement `GET /homilies` within `HomiliesController` mapping to `HomiliesService.findAll(query)`. The endpoint must support optional pagination (`limit`, `page`) and a text-based `search` query parameter using Prisma's `contains` filter on the title/content fields. The payload should return metadata devoid of heavy content fields if possible or the full list if simple.
</action>
<acceptance_criteria>
- `homilies.controller.ts` contains `@Get()` mapped to a method returning a list
- `homilies.service.ts` contains a Prisma `.findMany(` call utilizing `contains:` when search is present
</acceptance_criteria>
</task>

<task>
<read_first>
- apps/api/src/homilies/homilies.controller.ts
- apps/api/src/homilies/homilies.service.ts
</read_first>
<action>
Implement `GET /homilies/:slug` mapping to `HomiliesService.findBySlug(slug)`. This method securely fetches the single `Homily` record via Prisma targeting `where: { slug }`. It returns a 404 Exception if the record doesn't exist.
</action>
<acceptance_criteria>
- `homilies.controller.ts` contains `@Get(':slug')`
- `homilies.service.ts` contains `.findUnique({ where: { slug } })`
</acceptance_criteria>
</task>

## Verification
- Code passes local `nest build` without Type errors.
- The endpoints can be mapped.

## Must Haves
- The Next.js frontend will have sufficient query mechanisms (search param) to build a robust dashboard.
- CORS is enabled so Traefik/Next.js client-side can fetch via browser securely.
